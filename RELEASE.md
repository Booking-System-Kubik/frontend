## 3) Scope релиза

### Входит в релиз

- Авторизация и проверка сессии.
- Базовые сценарии бронирования (поиск, создание, отмена).
- Админские операции по рабочим пространствам (в текущем реализованном объеме).
- Базовая аналитика (в текущем реализованном объеме).
- Инструкции по запуску через Docker Compose.

### Не входит в релиз

- Полноценный production-grade CI/CD.
- Высоконагруженное тестирование и SLA.
- Выделенная внешняя DevOps-инфраструктура.

## 4) Инструкция по развертыванию (финальная версия)

1. Подготовить `.env` для backend (DB/JWT/admin-параметры).
2. Запустить backend: `docker compose up --build -d` из каталога `backend` (поднимаются только `app` и `postgres`). Мониторинг (Prometheus/Grafana) по желанию: `docker compose --profile monitoring up --build -d`.
3. Запустить frontend (`npm install`, `npm run dev`) с корректным `NEXT_PUBLIC_API_URL`.
4. Проверить доступность:
   - backend: `http://localhost:8080/actuator/health`
   - frontend: `http://localhost:3000`

Результат проверки: запуск воспроизводим на чистом окружении при соблюдении инструкции.

## 5) Безопасность (обязательный раздел)

## 5.1 Модель угроз (минимум)

- Утечка секретов (`JWT_SECRET`, пароли БД, admin credentials).
- Публикация внутренних сервисов наружу (БД/мониторинг).
- Слабые/дефолтные пароли и тестовые учетные данные.
- Избыточные права у пользователей/администраторов.

## 5.2 Статус безопасности к релизу

- Управление секретами переведено в переменные окружения, чувствительные данные в репозиторий не попадают.
- Контур контейнеризации проверен: приложение и база данных поднимаются в штатной конфигурации.
- Базовая ревизия API и ролей выполнена, критичных блокирующих рисков к релизу не выявлено.

- [x] Реальные секреты не хранятся в репозитории (`backend/.env` исключен из git).
- [x] Переменные окружения для запуска backend вынесены в `.env` (DB/JWT/admin-параметры).
- [x] Подтвержден рабочий локальный запуск backend через Docker Compose.
- [x] Проверена ролевая модель и доступ к административным эндпоинтам.

### 5.4 Что уже сделано по угрозе утечки секретов

- Реальные секреты не хранятся в репозитории: в `backend/.gitignore` исключен `backend/.env`.
- Для демонстрации и релиза используются локальные/закрытые значения переменных окружения.
- Команда использует отдельный `.env` для локального запуска и не публикует его содержимое.
